# Data Sources

## Overview

All data in this project is **synthetic** - generated locally by
`scripts/generate_synthetic_data.py`. No real API keys, business
accounts, or external services are required to run this project.

This is a deliberate design decision (see ADR-004) - real Shopify and
Amazon Selling Partner API access requires business verification and app
review, not obtainable within a 5-week internship window. The pipeline
engineering (schema handling, deduplication, partitioning, quality checks)
is identical regardless of whether the data is real or synthetic.

## Source 1: Shopify Orders API (mock)

| Field | Value |
|---|---|
| **Description** | E-commerce order transactions from CartCo's storefront |
| **Format** | CSV (generated), ingested as Delta Lake |
| **Volume** | ~510 rows (500 base + ~2% duplicates injected) |
| **License** | N/A - synthetic, generated by this project |
| **Schema** | See [docs/sources/sources.md](sources/sources.md) |
| **Location** | `scripts/output/shopify_orders.csv` (gitignored) |
| **Sample** | `scripts/data/samples/shopify_orders_sample.csv` (20 rows, tracked) |
| **Data Contract** | `docs/contracts/shopify_orders_contract.yaml` |

**Deliberate quality issues injected:**
- ~3% null `order_status` values - simulates incomplete upstream records
- ~2% duplicate rows - simulates API retry/resend behavior
- ~2% late-arriving records - records whose `order_date` is 1-3 days before the ingestion date
- ~1% rows with an unexpected `promo_code_applied` column - simulates upstream schema drift

## Why synthetic data?

The B1 problem statement explicitly permits synthetic data: *"synthetic
data is fine."* What's evaluated is pipeline engineering skill, not data
authenticity. Generating synthetic data with deliberate, realistic
messiness is arguably a stronger demonstration - it shows understanding
of real data quality problems, not just the ability to pass clean data
through a pipeline.

## Data generation

Reproducible via:

```bash
python scripts/generate_synthetic_data.py
```

Generation is seeded (`SEED = 42`) for consistent shape across runs.
Full documentation: [docs/data_generation.md](data_generation.md)
